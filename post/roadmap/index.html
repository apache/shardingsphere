未来线路规划

进行中

config

  1. Spring命名空间
  2. yaml配置
  3. 动态配置逻辑表，例：logictables=“order${0..64}”
  4. 分片算法inline支持，例：sharding_express=“user_id % 8”

transaction

  1. sharding-jdbc-core发送SQL执行事件
  2. bed类型事务日志存储设计
  3. bed类型事务同步尝试
  4. 基于elastic-job的bed类型事务异步尝试

计划中

config 2.0

  1. 动态表配置
  2. 只分库不需要逻辑表和真实表对应配置
  3. 简化只分库配置，无需配置逻辑表和真实表对应关系
  4. 简化只分表配置，可指定默认数据源,简化单库TableRule配置

transaction 2.0

  1. 基于tcc的柔性事务

distribution id

  1. JDBC接口改写
  2. 分布式主键策略接口制定
  3. 基于snowflake的分布式主键算法实现
  4. 基于groupsequence的分布式主键算法实现
  5. 基于数据库的分布式主键算法实现

parser 2.0

  1. 判断不支持SQL并直接报错
  2. 支持DISTINCT
  3. 支持GROUP BY聚合之后进行HAVING
  4. 支持计算表达式，如：SUM(pv) / COUNT(uv)
  5. 支持通过SQL注释指定SQL Hint
  6. SQL重写优化，进一步提升性能

merger 2.0

  1. 管道化结果归并
  2. 支持OR语句根据row的唯一标识去重
  3. 支持DISTINCT
  4. 支持GROUP BY聚合之后进行HAVING
  5. 支持计算表达式，如：SUM(pv) / COUNT(uv)

router 2.0

  1. 支持通过SQL注释指定SQL Hint
  2. ThreadLocal Hints需重新考虑生命周期是否和connection以及statement绑定
  3. 释放资源时，自动清理ThreadLocal Hints

待定

  1. 读写分离
  2. 字典表复制广播
  3. HA相关
  4. 流量控制
  5. 建表工具
  6. 动态扩容
  7. 配置中心
  8. 其他数据库支持
